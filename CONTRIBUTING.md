# Contributing to BlissCampIndia

Thank you for your interest in contributing to **BlissCampIndia**! We welcome contributions from the community to improve and enhance this project. Please follow the guidelines below to ensure a smooth collaboration.

## How to Contribute

### 1. Reporting Issues
If you find a bug or have a feature request, please open an issue:
- Go to the [Issues](https://github.com/NK2552003/BlissCampIndia/issues) tab.
- Click on **New Issue**.
- Provide a clear and detailed description of the issue.
- If applicable, include screenshots or code snippets to help illustrate the problem.

### 2. Forking the Repository
To contribute to the project, you need to fork the repository and create a local copy:
1. Click the **Fork** button on the BlissCampIndia repository.
2. Clone your forked repository:
   ```bash
   git clone https://github.com/your-username/BlissCampIndia.git
   ```
3. Navigate to the project directory:
   ```bash
   cd BlissCampIndia
   ```
4. Add the original repository as an upstream remote:
   ```bash
   git remote add upstream https://github.com/NK2552003/BlissCampIndia.git
   ```

### 3. Creating a Branch
Before making changes, create a new branch:
```bash
git checkout -b feature-branch-name
```
Use a meaningful branch name related to your changes (e.g., `fix-header-bug` or `add-new-component`).

### 4. Making Changes
- Follow the coding style used in the project.
- Ensure your changes do not break existing functionality.
- Test your changes before committing.
- If adding a new feature, update documentation accordingly.

### 5. Committing Changes
Write clear and descriptive commit messages:
```bash
git commit -m "Fix: corrected layout issue in header"
```
Follow conventional commit messages when possible (e.g., `Fix:`, `Add:`, `Update:`).

### 6. Pushing Changes
Push your branch to your forked repository:
```bash
git push origin feature-branch-name
```

### 7. Creating a Pull Request
1. Go to the original repository: [BlissCampIndia](https://github.com/NK2552003/BlissCampIndia/)
2. Click on **Pull Requests** and then **New Pull Request**.
3. Select your forked branch and compare it with the `main` branch.
4. Provide a clear title and description for your PR.
5. Submit the pull request for review.

## Code of Conduct
Please follow our [Code of Conduct](CODE_OF_CONDUCT.md) to ensure a positive and respectful environment for everyone contributing.

## Need Help?
If you have any questions, feel free to open an issue or reach out to the maintainers.
